# UI設計

## デザインコンセプト

シンプルで見やすく、100日チャレンジの継続感が伝わるUIにする。個人開発らしい素朴さを残しながら、ポートフォリオとして信頼できる見た目を目指す。

## トーン

- 落ち着いた背景色
- 読みやすい本文
- Day番号や進捗でチャレンジ感を出す
- 装飾よりも情報の見つけやすさを優先する
- 過度なグラデーションや派手なアニメーションは避ける

## レイアウト

### 全体

- 最大幅を設けた中央寄せレイアウト
- セクションごとに十分な上下余白
- モバイルでは1カラム
- タブレット以上ではカードを2から3カラムにする

### ファーストビュー

表示要素:

- タイトル: `1日1アプリ 100日チャレンジ`
- サブコピー: `普通の会社員が、AI時代に置いて行かれないように100日間アプリを作る記録`
- 現在の進捗: `Day x / 100`
- 進捗バー
- 主要リンクがある場合は GitHub や最新アプリへの導線

デザイン:

- ヒーローとして十分な存在感を出す
- テキスト主体にし、進捗バーを視覚的な主役にする
- ファーストビュー内でアプリ一覧の一部が少し見える余白設計にする

### 最新アプリ

- 一番新しい `published` または最新Dayのアプリを強調表示する。
- 通常カードより少し広いレイアウトにして、現在進行中の記録として見せる。
- アプリURL、GitHub URL が存在する場合はボタンを表示する。

### アプリ一覧

カードに表示する内容:

- Day番号
- タイトル
- 概要
- 技術タグ
- ステータス
- 詳細ページへのリンク

カードの状態:

- `published`: 公開済みとして通常表示
- `building`: 制作中として少し目立つラベル
- `planned`: 予定として控えめに表示

### アプリ詳細

- 冒頭に Day番号、タイトル、概要を配置する。
- スクリーンショットは存在する場合のみ表示する。
- 本文項目は「作った理由」「苦労した点」「学び」をブロック分けする。
- 前後リンクはページ下部に配置する。

## 色

初期案:

- Background: `#fafaf9` 付近
- Surface: `#ffffff`
- Text: `#1f2937` 付近
- Muted: `#6b7280` 付近
- Accent: 緑、青、または琥珀系のうち1色を控えめに使う

単一色に寄りすぎないよう、背景、テキスト、アクセント、タグ色に役割を分ける。

## タイポグラフィ

- 日本語の読みやすさを優先する。
- `next/font` を使う場合は既存の Geist を継続してもよい。
- 見出しは大きくしすぎず、本文との階層差を明確にする。
- 文字間隔は通常値を基本にする。

## レスポンシブ

### Mobile

- 1カラム
- カード幅は画面幅に追従
- ボタンは押しやすい高さにする
- タグは折り返し可能にする

### Tablet / Desktop

- 一覧は2から3カラム
- 詳細ページは本文幅を読みやすく制限する
- ヒーローと最新アプリは横幅を広めに使う

## アクセシビリティ

- リンクテキストは遷移先が分かる文言にする。
- ステータスは色だけでなくテキストでも表現する。
- 画像には内容が分かる `alt` を設定する。
- フォーカスリングを消さない。
- コントラスト不足の薄い文字色を避ける。
