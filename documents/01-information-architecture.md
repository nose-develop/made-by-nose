# 情報設計

## サイト全体の情報構造

このサイトは、チャレンジの文脈を伝えるトップページと、各アプリの制作記録を残す詳細ページで構成する。

```text
/
└── apps/[slug]
```

## トップページで伝えること

トップページは「何のサイトか」「いま何日目か」「どんなアプリを作っているか」を短時間で把握できる構成にする。

表示順:

1. ファーストビュー
2. 最新アプリ
3. アプリ一覧
4. チャレンジの説明
5. プロフィール / 運営者情報
6. 外部リンク

## アプリ詳細ページで伝えること

詳細ページは、単なるリンク集ではなく制作ログとして読める構成にする。

表示項目:

- アプリ名
- Day番号
- 公開日
- ステータス
- 概要
- スクリーンショット
- 使用技術
- 作った理由
- 苦労した点
- 学び
- アプリURL
- GitHub URL
- 前後のアプリへのリンク

## コンテンツ優先度

### 高

- Day番号
- アプリ名
- 概要
- 進捗
- アプリURL
- GitHub URL

### 中

- 技術タグ
- 制作理由
- 苦労した点
- 学び
- スクリーンショット

### 低

- 外部SNSリンク
- プロフィール詳細
- 補足的な説明文

## 100件まで増えた場合の考え方

初期実装ではページネーションや検索は入れない。ただし、将来追加しやすいようにデータ取得ユーティリティを用意し、UIではカードグリッドを使う。

将来の拡張候補:

- ステータス絞り込み
- 技術タグ絞り込み
- 年月別アーカイブ
- Day範囲でのジャンプ
- 一覧ページの独立
