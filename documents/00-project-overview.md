# プロジェクト設計概要

## プロジェクト名

made-by-nose

## 目的

「普通の会社員が、AI時代に置いて行かれないように100日間アプリを作る記録」として、1日1アプリ100日チャレンジで作成したアプリを継続的に掲載するポートフォリオサイトを作る。

## 想定URL

- Production: `https://www.madebynose.com/`
- Deploy: Vercel
- Source: GitHub push による Vercel 自動デプロイ

## 技術スタック

- Next.js `16.2.6`
- React `19.2.4`
- TypeScript
- Tailwind CSS `4`
- App Router
- 静的データ管理

## 設計方針

- 最初は DB / CMS / 管理画面を使わず、`src/data/apps.ts` の静的配列を単一の情報源にする。
- 2日目以降は `src/data/apps.ts` に1件追加するだけでトップページと詳細ページに反映される構成にする。
- 100件まで増えても一覧性を保つため、カード型グリッド、ステータス表示、技術タグ、進捗表示を基本UIにする。
- 個人開発の温度感を残しつつ、ポートフォリオとして見やすい余白、文字サイズ、色数に抑える。
- App Router の Server Components を基本にし、必要な場合だけ Client Components を切り出す。

## スコープ

### 初期実装に含める

- トップページ `/`
- アプリ詳細ページ `/apps/[slug]`
- 404ページ
- 共通ヘッダー
- 共通フッター
- アプリカード
- 進捗バー
- 静的データと取得ユーティリティ
- SEOメタデータ
- OGP基本設定
- README更新

### 初期実装に含めない

- DB
- CMS
- 管理画面
- 認証
- コメント機能
- 検索機能
- 外部APIからのデータ取得
- 動的OGP画像生成

## 成功条件

- `npm run build` が成功する。
- Day1 の「1日1アプリ公開サイト」がトップページと詳細ページで表示される。
- `src/data/apps.ts` に Day2 以降を追加すると、一覧、最新アプリ、詳細ページ、前後リンクに反映される。
- スマホ幅でもカード、進捗、詳細情報が読みやすい。
- Vercel で追加設定なしにデプロイできる。
